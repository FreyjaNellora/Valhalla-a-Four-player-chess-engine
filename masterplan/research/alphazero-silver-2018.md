# AlphaZero — Self-Play Co-Development

**Citation:** Silver, D. et al. (2018). "A general reinforcement learning algorithm that masters chess, shogi, and Go through self-play." *Science*, 362(6419):1140-1144.
**URLs:**
- https://www.science.org/doi/10.1126/science.aar6404
- https://arxiv.org/abs/1712.01815

## How the Self-Play Cycle Works

1. Generate self-play games using current network + MCTS (800 sims/move)
2. Store (state, MCTS policy, game outcome) tuples in replay buffer (1M most recent games)
3. Train network on sampled batches to minimize combined loss
4. Repeat with improved network

## Training Target

**Loss function:** L = (z - v)² - πᵀ·log(p) + c||θ||²
- (z - v)²: MSE between predicted value v and actual game outcome z
- -πᵀ·log(p): Cross-entropy between MCTS search policy π and network policy p
- Regularization term

**Critical detail:** Value targets come from actual game outcomes (terminal states), NOT bootstrapped intermediate estimates. This provides stable learning.

## MCTS Integration

- Single network with two heads: policy (move probabilities) and value (expected outcome)
- At each MCTS leaf, network provides (policy prior, value estimate)
- Policy prior biases exploration (PUCT formula)
- Value estimate backed up through tree

## Scale

- 5000 TPUs for self-play, 16 TPUs for training
- SGD, momentum 0.9, batch 4096, LR 0.2 (dropped 3x)
- Chess: outperformed Stockfish after 4 hours (300k steps)

## Relevance to Valhalla

The search-evaluation co-development loop is Valhalla's core principle. NNUE (apprentice) learns from self-play data generated by OPPS+Swarm (expert). The trained NNUE improves OPPS evaluation, generating better training data. The key insight: train on search policy (what moves search found best), not just game outcome.
